Configgy 是常駐選單列的 macOS app：把 Zen Browser、Claude Code、以及任何你指定的資料夾，存成帶日期的 zip 快照放進你的雲端資料夾——可預覽、可回滾、換機器一鍵還原。密碼與私鑰一概不碰。

運作原理

一個目標，就是一組路徑

Configgy 只有一個模型：把一組路徑存成帶日期的 zip、留著歷史、還原前先給你看 diff。Zen、Claude Code、你自己加的資料夾，走的都是同一條路。

1

選好目標

預設只開 Claude Code。Zen 和其他都是「建議」，你點加入才開始版本化——這台沒裝 Zen 也照常跑。

2

存成版本化 zip

快照落在 Dropbox / iCloud / Google 雲端硬碟 / 自選資料夾 的 …/Configgy/；留 10 份、去重，檔名帶主機名，兩台 Mac 不會打架。

3

回得去才算數

還原前先列出哪些檔案會變、覆寫前把舊檔備起來。挑舊快照就是回滾；Zen 的 zip 內嵌 restore.sh，裸機也還原得回來。

不碰機密：密碼、cookie、瀏覽紀錄、SSH 私鑰、含 token 的 gh hosts.yml 一律不複製——那些該靠你的 Mozilla 帳號和鑰匙圈回來。探索也只建議「本身沒有雲端同步」的設定。

功能

顧的是設定，不是整顆家目錄

那些弄丟很痛、又不適合整包丟雲端同步的東西，剛好就是這個 app 的守備範圍。

🌀

Zen 關掉就備份

關閉 Zen 時自動打包 profile；也能只還原選定的 workspace。要它現在備份，會再問一次才關瀏覽器。

✳︎

Claude Code 設定

快照 ~/.claude 與 ~/.agents/skills；還原時連 plugins 一起重裝，新機器接得回原本的手感。

🔍

自動探索建議

掃出這台 Mac 上沒有雲端同步的設定：shell、git、~/.ssh/config、Zed、VS Code、終端機、Karabiner、Hammerspoon…點加入就收編。

↩︎

預覽、回滾、再納入

還原先看 diff；每個目標都有歷史可挑。開啟時掃一遍備份資料夾，另一台 Mac 備過的目標會自己回到清單上。

🎛️

一個視窗就夠

膠囊 tab 切兩份清單，動作就在列上；設定是視窗內的一頁，不是層層選單。淺色深色都是一等公民。

⌨︎

同一顆 binary 就是 CLI

Configgy backup、list、restore、target-add…全部跑得動，排程或遠端都行。

畫面

長這樣

已備份保護 — 你正在保護的目標。每列：app 圖示、最後備份 · N 份快照，動作就在同一列上。

淺色模式 — 同一個視窗換上 Finder 風的淺色配色；系統外觀一切換，顏色就重新解析。

建議加入 — 這台 Mac 上沒有雲端同步的設定（shell、git、不含私鑰的 SSH config、tmux、Zed…）。點加入就開始版本化。

設定 — 在視窗裡，不在深層選單：主題、放大介面、暫停 Zen、開機自動啟動、語言、備份位置、關於。

安裝

下載 dmg，拖進「應用程式」

自簽 App，第一次打開請用右鍵 → 打開；或跑下面這行清掉 quarantine。第一次啟動時 app 會帶你把權限開好、選好備份位置。

# 清除 quarantine 後直接打開
xattr -dr com.apple.quarantine /Applications/Configgy.app
open /Applications/Configgy.app

完整磁碟取用權限：Configgy 要讀受保護的設定資料夾、寫進 Dropbox 之類的雲端資料夾，這兩件事 macOS 都擋。偏偏系統不會自己跳出請求，所以請到「系統設定 → 隱私權與安全性 → 完整磁碟取用權限」把 Configgy 打開（app 首次啟動也會引導你）。沒授權它就碰不到備份資料夾。

寫在最後

一開始它叫 Zennly，只會做一件事：Zen 一關就把 profile 壓成 zip 丟 Dropbox。後來 Claude Code 的設定也想備份，才發現需要的不是第二個 app，而是一個模型——一組路徑、版本化的 zip、有歷史、有 diff、還原是加上去而不是蓋掉。

Configgy 很小、開源、免費，是自己要用才寫的工具。整個專案是用 Claude Code 邊聊邊長出來的，介面靠著看截圖一輪輪調。如果它幫你在換機器那天少痛一點，那就值得了。
